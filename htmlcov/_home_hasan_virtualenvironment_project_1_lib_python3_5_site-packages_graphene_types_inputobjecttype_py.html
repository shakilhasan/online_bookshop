33 """ 

34 Input Object Type Definition 

35 

36 An input object defines a structured collection of fields which may be 

37 supplied to a field argument. 

38 

39 Using ``graphene.NonNull`` will ensure that a input value must be provided by the query. 

40 

41 All class attributes of ``graphene.InputObjectType`` are implicitly mounted as InputField 

42 using the below Meta class options. 

43 

44 .. code:: python 

45 

46 from graphene import InputObjectType, String, InputField 

47 

48 class Person(InputObjectType): 

49 # implicitly mounted as Input Field 

50 first_name = String(required=True) 

51 # explicitly mounted as Input Field 

52 last_name = InputField(String, description="Surname") 

53 

54 The fields on an input object type can themselves refer to input object types, but you can't 

55 mix input and output types in your schema. 

56 

57 Meta class options (optional): 

58 name (str): the name of the GraphQL type (must be unique in schema). Defaults to class 

59 name. 

60 description (str): the description of the GraphQL type in the schema. Defaults to class 

61 docstring. 

62 container (class): A class reference for a value object that allows for 

63 attribute initialization and access. Default InputObjectTypeContainer. 

64 fields (Dict[str, graphene.InputField]): Dictionary of field name to InputField. Not 

65 recommended to use (prefer class attributes).
